## 1. 快速上手：10分钟写出你的第一章

假设你已经部署好系统并打开了网页，按下面的流程走一遍：

1. **注册/登录**：输入用户名和密码（密码至少8位、包含字母和数字）
2. **新建项目**：进入首页"总控驾驶舱"，点击"新建项目"按钮，在弹窗里填：
   - 书名（必填）
   - 类型（下拉选择，比如玄幻、都市、悬疑等）
   - 目标平台（下拉选择你打算发布的平台）
3. **进入创作工作台**：点击刚创建的项目卡片，进入写作页面
4. **生成第一章**：点击章节列表底部的"新章生成"按钮，等待 AI 生成（通常需要 1~3 分钟，取决于 AI 服务响应速度）
5. **阅读与修改**：生成完成后正文出现在中间的编辑器里，你可以直接阅读，也可以点击编辑按钮手动修改
6. **继续生成下一章**：再点一次"新章生成"，AI 会自动带着前面章节的记忆继续往下写

就这么简单。系统在背后自动做了很多事（组装上下文、记录人物状态、追踪伏笔、质量审查），这些后面章节会详细讲。

## 2. 界面总览：8个页面分别是干什么的

登录后，屏幕左侧（手机上是底部）是导航栏，共 8 个入口：

| 导航项 | 干什么用 | 什么时候用 |
|---|---|---|
| **总控驾驶舱** | 首页。所有作品的卡片列表、快捷操作入口、后台任务队列状态 | 每次打开系统的起点 |
| **创作工作台** | 核心写作页面：章节列表、正文编辑器、AI 工具、上下文面板 | 写作时绝大部分时间在这里 |
| **伏笔看板** | 管理这本书埋下的所有伏笔：待回收/已回收/超期 | 长篇写作中期定期来看 |
| **质量面板** | 每一章的 7 维质量评分、问题列表、AI 定向重写 | 每写几章来检查一次质量 |
| **翻译发布** | 把章节翻译成适配海外平台风格的英文 | 想把作品发到海外平台时 |
| **爆款分析** | 输入书名/题材/开头，AI 分析市场契合度、给出选题建议 | 开新书之前 |
| **数据分析** | 产出统计、趋势图、各项目对比 | 想回顾自己写作数据时 |
| **设置** | API Key 配置、Token 用量、账户信息、暗色模式 | 首次使用时配置一次 |

> 注意：**创作工作台、伏笔看板、质量面板、翻译发布**这4个页面是"跟着某本书走"的——你需要先在首页选中一个项目，这些页面才有内容可显示。

## 3. 登录与账号

- **注册**：登录页上方有"登录/注册"两个标签，切换到"注册"，填写用户名、邮箱、密码、确认密码。密码要求至少 8 位且同时包含字母和数字。
- **登录状态保持**：系统使用安全的 httpOnly Cookie 机制，登录后短时间内无需重复登录；令牌过期时系统会自动静默续期，你不会感觉到中断。长时间（超过 7 天）不使用后需要重新登录。
- **登出**：设置页面底部有"退出登录"按钮。
- **管理员账号**：部署时在 `.env` 里配置的 `ADMIN_USERNAME`/`ADMIN_PASSWORD` 会在系统首次启动时自动创建，功能上与普通账号一致。
- **多用户**：系统支持多个用户各自注册使用，每个人的作品、章节、伏笔等数据完全隔离，互相看不到。

## 4. 总控驾驶舱（首页）

首页由三部分组成：

### 4.1 项目卡片区
每个项目一张卡片，显示书名、类型、章节数、总字数。
- **点击卡片** → 进入该项目的创作工作台
- **新建项目** → 打开创建弹窗（书名 + 类型 + 目标平台）

### 4.2 快捷操作区
四个快捷入口：
- **新建项目**：同上
- **扫榜分析**：跳转到爆款分析页面
- **批量生成**：当前版本此按钮尚未接通实际功能（批量生成的后端能力完整，发起方式见第 6 节）
- **拆文学习**：跳转到爆款分析页面（拆文分析能力目前整合在该页面中）

### 4.3 任务队列区
显示后台正在执行/排队中的任务（批量生成、质量审查等）的实时状态和进度。批量生成发起后，你可以离开页面做别的事，回到首页就能看到进度。

## 5. 创作工作台：核心写作页面

这个页面分为三栏（手机上会自动折叠成单栏切换）：

### 5.1 左栏：章节导航
- 顶部是项目信息（书名、类型、总字数）
- 中间是章节树：按章节号排列，每章显示标题、字数、质量评分、状态（草稿/已审查）
- **点击任意章节** → 中间编辑器切换到该章内容
- 底部是**"新章生成"按钮**：点击后 AI 自动续写下一章

### 5.2 中栏：正文编辑器
- 顶部信息栏：当前章节标题、字数，以及一个**编辑/只读切换按钮**（铅笔图标）——默认只读防止误触，想手动改内容时点它切换到编辑模式
- 正文区域：舒适的阅读排版；处于编辑模式时可以直接修改文字
- **AI 工具栏**提供四个操作：
  | 按钮 | 作用 | 说明 |
  |---|---|---|
  | 续写 | 让 AI 接着当前进度生成下一章 | 与左栏"新章生成"效果相同 |
  | 优化改写 | 对当前整章做一次整体表达优化 | 保留情节，只改善文笔；改写前系统会自动保存当前版本快照，可回退 |
  | 去AI味 | 消除文本中明显的"AI腔"表达 | 针对"总而言之""值得注意的是"这类机器味措辞 |
  | 重新审查 | 对当前章节重新跑一次 7 维质量审查 | 结果在质量面板查看，评分也会更新到右下角的评分球 |

### 5.3 右栏：上下文面板
展示 AI 在生成这一章时"记得"哪些东西：全书总纲、最近章节摘要、人物状态、待回收伏笔等。这个面板是只读的展示区，帮你确认 AI 的记忆是否符合预期。

### 5.4 右下角：质量评分球
一个悬浮的圆形小球，显示当前章节最近一次审查的综合评分（0-100）。没审查过时显示"暂无"。点击可快速跳转到质量面板。

### 5.5 AI 是怎么"记住"前文的？
每次生成新章节，系统自动组装一个多层上下文注入给 AI：
1. 全书总纲
2. 世界观设定与规则
3. 人物当前状态（每章生成后自动更新）
4. 最近 5 章的摘要（每章生成时自动提炼 150 字左右摘要存档）
5. 当前待回收的伏笔列表
6. 防崩提醒（近期审查发现的一致性问题）

所以你不需要手动告诉 AI"上一章发生了什么"——但如果你**手动大幅修改了某章内容**，建议对该章点一次"重新审查"，让系统的摘要和记忆与你的修改保持同步。

## 6. 批量生成：让 AI 自动连续写多章

### 6.1 当前版本的真实状态（重要，请先读这一段）
批量生成的**后端能力是完整且经过验证的**（严格顺序生成、自动审查、失败正确停止），但**当前版本的网页界面上还没有接好发起批量生成的按钮**——首页的"批量生成"快捷入口目前点击后没有实际动作。这是一个已知的待完善项。

在界面入口补上之前，如果你需要使用批量生成，可以通过 API 直接发起（把下面的 `<项目ID>` 换成你的项目 ID，项目 ID 可以在浏览器地址栏里看到，进入创作工作台后地址是 `/write/<项目ID>` 这样的格式）：

```bash
curl -X POST http://localhost:8100/api/v1/pipeline/batch-generate \
  -H "Content-Type: application/json" \
  -b "access_token=<你的登录token>" \
  -d '{"project_id": "<项目ID>", "chapter_count": 5}'
```

> 更简单的方式：用浏览器打开 `http://localhost:8100/docs`，这是系统自带的交互式 API 文档页面，找到 `POST /api/v1/pipeline/batch-generate`，点 "Try it out" 直接在网页里填参数执行（需要先在同一个浏览器里登录过系统）。

发起后回到首页的任务队列区就能看到进度。

### 6.2 它是怎么工作的（了解这个能帮你正确使用）
- **严格按顺序逐章生成**：第 N+1 章一定会等第 N 章完整写完、入库之后才开始，保证每一章都能读到完整的前文记忆，不会出现"后一章不知道前一章写了什么"的情况。
- **每章生成后自动触发质量审查**：不需要你手动逐章点审查。
- **失败自动处理**：AI 服务临时不可用时系统会自动重试；如果遇到无法恢复的问题（比如 Token 预算用完），整个批次会停止并在任务队列里标记失败原因，**不会带着错误继续往下写**。
- **进度实时可见**：回到首页任务队列区就能看到"已完成 X / 共 Y 章"。

### 6.3 使用建议
- 第一次使用建议先批量生成 3~5 章看看效果，满意再加大批量。
- 批量生成期间不建议同时手动编辑同一本书的章节，等批次跑完再改。
- 每章生成会消耗 DeepSeek Token（约几千到一万 Token/章），批量前留意账户余额。

## 7. 质量面板：7维评分与 AI 自动改写

### 7.1 七个维度
每次审查会从以下 7 个维度给章节打分（每维 0-10 分）+ 综合分（0-100）：

| 维度 | 检查什么 |
|---|---|
| 一致性 | 人物、时间线、地点、设定是否与前文矛盾 |
| AI味检测 | 是否有明显的机器腔表达 |
| 节奏 | 情节推进是否拖沓或过快 |
| 人物OOC | 角色言行是否符合已建立的人设 |
| 爽点密度 | 阅读快感点的分布是否合理 |
| 对话质量 | 对话是否自然、有信息量、有张力 |
| 结尾钩子 | 章末是否留下让读者想追下一章的悬念 |

### 7.2 页面内容
- 综合评分环、各维度得分卡片（附具体问题列表）、全部章节的评分表格
- **AI 定向重写**：对某个低分维度点击重写，AI 会针对该维度的问题改写，页面上以**左右对比**的形式展示原文与改写结果

### 7.3 自动改写机制（重要）
系统内置了自动质量闭环：**每次审查后，如果发现有维度得分低于 6 分，系统会自动针对得分最低的那个维度做一次整章改写，并直接应用到章节正文**。你需要知道的是：
- **改写前的原文永远不会丢**：系统在应用任何 AI 改写之前，都会先把当前版本存为历史快照，你可以随时对比差异、回退到任何历史版本。
- 每轮审查最多自动改写一个维度（得分最低的那个），不会多个维度反复改写把文章改乱。
- 如果你不希望某章被自动改写，可以在生成后先手动编辑定稿，改动后的内容再审查时如果分数合格就不会触发改写。

## 8. 伏笔看板：埋线与回收管理

长篇小说最容易出的问题就是"挖了坑忘了填"。这个页面帮你管理所有伏笔。

### 8.1 伏笔从哪来
AI 生成章节时会自动识别并登记两类信息：
- **本章新埋下的伏笔**（描述 + 埋设章节号 + 预期回收章节范围）
- **本章回收了哪些之前的伏笔**（自动把对应伏笔标记为"已回收"）

你不需要手动录入，但可以随时来这个页面查看和干预。

### 8.2 页面结构
三列看板 + 顶部统计：
- **待回收**：埋了还没填的坑
- **已回收**：填完的坑（显示埋设章节 → 回收章节）
- **超期**：超过了预期回收范围还没填的坑（系统每次生成新章节时自动检测标记）

支持按状态筛选、按关键词搜索。

### 8.3 手动操作
- **AI 检查回收**：对某条伏笔，粘贴回收它的章节正文，AI 会分析这次回收的质量（是否呼应到位、是否仓促）
- **标记回收**：手动把某条伏笔标记为已回收（同样需要提供对应章节正文供 AI 确认）
- **逾期检查**：手动触发一次全量超期检测

> 当前版本中，这两个手动操作通过一个简单的文本输入框收集章节正文，交互比较朴素，后续版本会升级为正式的选择弹窗。

## 9. 爆款分析：写之前先看市场

开新书之前，用这个页面让 AI 帮你评估选题。

### 9.1 输入
- 书名（构思中的名字）
- 类型
- 目标平台
- 第一章内容（可选，粘贴进去分析会更精准）

### 9.2 输出
- **爆款指数**（0-100）与市场契合度评价
- **题材趋势**：当前市场上该类型的热度信息
- **选题建议**：AI 基于分析给出的调整方向
- **市场预测**：对这个选题前景的判断

> 使用提示：AI 的市场分析是基于其训练知识和文本分析能力给出的参考意见，不是实时抓取的榜单数据，做重大决策时应结合你自己对目标平台的实际观察。

## 10. 翻译发布：出海翻译

把你的章节翻译成适配海外网文平台风格的英文。

### 10.1 页面结构
- **左侧**：章节选择列表 + 目标平台选择 + 术语表
- **右侧**：翻译结果展示区

### 10.2 支持的目标平台风格
| 平台 | 翻译风格 |
|---|---|
| Webnovel | 流畅自然，保留东方修炼体系术语，适合移动端阅读 |
| Royal Road | 文学性较强，LitRPG / Progression Fantasy 风格 |
| Wattpad | 青春化表达，对话比例高，段落简短 |
| Scribble Hub | 介于 Webnovel 和 Royal Road 之间，接受日式轻小说元素 |

### 10.3 术语表
翻译修炼小说最头疼的就是专有名词（功法、境界、门派名）翻译不统一。在左侧术语表里预先登记"中文术语 → 英文译名"的对应关系，AI 翻译时会严格遵守，保证全书译名一致。

### 10.4 操作流程
选择章节 → 选择目标平台 → （可选）维护术语表 → 点击翻译 → 右侧查看结果 → 复制去目标平台发布。

> 说明：当前版本的"发布"环节是手动的（复制译文自己去平台粘贴）。全自动登录发布功能的代码框架存在但尚未达到可用状态，见第 15 节。

## 13. 核心概念解释：系统是怎么防止AI写崩的

普通的"让 ChatGPT 帮我写小说"写到几万字就会崩：人物性格漂移、前后矛盾、忘记伏笔。本系统通过以下机制对抗这个问题，理解它们能帮你更好地使用：

1. **章节摘要记忆**：每章生成后自动提炼摘要存档，续写时注入最近 5 章摘要——AI 不需要重读全文也能"记得"最近发生了什么。
2. **人物状态追踪**：以结构化数据记录每个角色的当前状态（位置、目标、关系），每章自动更新，防止"上一章还在断崖底下，这一章突然出现在皇宫"。
3. **伏笔池**：所有埋设的伏笔集中登记，续写时把"待回收伏笔"注入上下文提醒 AI，超期未回收会被标记。
4. **7维审查 + 自动改写闭环**：每章生成后自动审查，低分维度自动改写，且改写前保留版本快照可回退。
5. **防崩提醒**：审查发现的一致性问题会作为警告注入后续章节的生成上下文，避免同一个错误滚雪球。
6. **严格顺序生成**：批量生成时后一章必定等前一章完全落库后才开始，从机制上杜绝"并行生成互相看不到对方内容"导致的断裂。

## 14. 使用建议与最佳实践

1. **新书先跑爆款分析**，用几毛钱的 Token 成本换一次选题体检。
2. **前三章亲自把关**：黄金三章决定留存，AI 生成后建议人工精修，你修改后的文风也会通过摘要影响后续生成。
3. **每 10 章看一次伏笔看板**，出现"超期"列堆积时，在后续章节里主动安排回收。
4. **善用版本快照**：AI 的任何自动改写都有历史版本可回退，大胆尝试"优化改写"，不满意就回退。
5. **控制成本**：批量生成前看一眼设置页的 Token 用量；单章生成约消耗几千到一万 Token，按 DeepSeek 的价格折算成本很低，但批量生成几百章时依然值得留意。
6. **定期备份**：按《部署文档》第 12 节定期备份数据库，你的心血都在里面。
