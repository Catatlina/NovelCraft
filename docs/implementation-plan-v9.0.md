> **原则**：每个功能都有真实按钮，点下去有真实效果，没有假接口

## Sprint 1：能跑通一条完整链路（5 天）

### 目标：灵感 → 第一章正文，4 个平台扫榜能用

| 任务 | 工作内容 | 产出 |
|------|---------|------|
| 1.1 扫榜爬虫 | `scanner.py` 用 httpx+bs4 爬 4 平台榜单 | GET /api/v1/scan/trending 返回真实榜单 |
| 1.2 灵感全链路 | 一句话→书名(3个)→简介→大纲→细纲→章节树→第一章 | POST /api/v1/projects/quick-start |
| 1.3 快速开始页 | **新页面** `/quick-start`，灵感输入+扫榜推荐+生成进度 | 一个页面走完整条链路 |
| 1.4 导航改造 | 侧边栏加「快速开始」按钮，仪表盘加醒目入口 | 所有页面能跳到这里 |
| 1.5 项目列表可达 | 已生成的书在仪表盘可看到、可点击 | 点项目→创作工作台 |

## 架构方案（无需改动现有后端结构）

```
现有不变（保留所有已工作的 53 个端点 + Celery + pgvector）

新增：
┌──────────────────────────────────┐
│ POST /api/v1/projects/quick-start │  ← 灵感全链路（Sprint 1）
│ GET  /api/v1/scan/trending        │  ← 真实爬虫（Sprint 1）
│ POST /api/v1/quality/{id}/targeted-rewrite │ ← 定向重写（Sprint 2）
│ POST /api/v1/foreshadows/{id}/check-payoff │ ← Payoff检测（Sprint 2）
│ POST /api/v1/projects/{id}/world-rules/check │ ← 规则校验（Sprint 3）
└──────────────────────────────────┘

前端新增：
┌──────────────────────────────┐
│ /quick-start      快速开始页  │
│ /admin/prompts     Prompt管理 │
└──────────────────────────────┘

前端重构（按钮全部接线，无假按钮）：
- Dashboard.tsx: 4 个快捷操作按钮→全部对接
- WritingWorkspace.tsx: 所有工具栏按钮→全部对接
- ForeshadowBoard: 筛选/埋点/回收/Payoff检测
- QualityDashboard: 审查/定向重写/历史对比
- TranslatePage: 翻译/预览/发布/历史
- TrendAnalysis: 表单/趋势/卡片/预测（从扫榜数据驱动）
- SearchBar: 全局搜索已可用
```
