## 1. 这是什么，部署完你会得到什么

星禾写作助手是一个 AI 驱动的网络小说写作平台，部署完成后，你会得到：

- 一个可以在浏览器里打开的写作网站（自己电脑上用，或者放到服务器上给别人用都可以）
- AI 自动续写章节、批量生成、质量审查、自动改写、伏笔追踪、翻译出海等一整套写作辅助功能

整套系统由 4 个部分组成，全部通过 Docker 一键启动，你不需要手动安装 Python、Node.js、数据库这些东西：

| 部分 | 作用 |
|---|---|
| 后端 API (api) | 处理所有业务逻辑，调用 AI 模型 |
| 后台任务队列 (worker) | 处理批量生成、质量审查这类耗时任务 |
| 数据库 (db) | 存放你的小说、章节、账号等所有数据 |
| 缓存/队列中间件 (redis) | 配合 worker 处理任务队列 |

你只需要装一个 Docker，剩下的这 4 个部分会被自动装好、自动启动。

## 2. 准备工作：你需要什么

### 2.1 硬件/系统要求
- 一台电脑或云服务器，Windows / macOS / Linux 都可以
- 至少 4GB 内存（推荐 8GB 以上），至少 10GB 剩余磁盘空间
- 能连接互联网

### 2.2 你需要提前准备好的账号
只有一个是必须的：

- **DeepSeek API Key**（必须）：这是驱动所有 AI 写作功能的模型服务商。
  1. 打开浏览器访问 `https://platform.deepseek.com`
  2. 注册一个账号（手机号或邮箱都可以）
  3. 登录后找到左侧菜单的 "API Keys"（API 密钥）
  4. 点击创建一个新的 Key，会显示一长串以 `sk-` 开头的字符
  5. **立刻复制保存下来**（很多平台这个 Key 只会完整显示一次，关掉页面就看不到完整内容了）
  6. 账号里需要绑定支付方式并预充值一点余额（几块钱人民币就可以体验很久），否则调用 AI 会失败

没有这个 Key，系统能打开、能注册登录，但所有跟 AI 生成/审查/翻译相关的功能都用不了。

## 3. 第一步：安装 Docker

Docker 是一个能让"这套系统在任何电脑上都长得一模一样"的工具，装好它之后，后面的步骤只需要敲几行命令。

### 3.1 Windows 用户
1. 打开浏览器访问 `https://www.docker.com/products/docker-desktop/`
2. 点击 "Download for Windows"，下载安装包
3. 双击安装包，一路点 "下一步"，安装完成后**重启电脑**
4. 重启后，在开始菜单搜索 "Docker Desktop"，打开它
5. 第一次打开可能需要等待一两分钟初始化，右下角状态栏图标从"转圈"变成一只固定不动的小鲸鱼图标，说明启动成功

### 3.2 macOS 用户
1. 打开浏览器访问 `https://www.docker.com/products/docker-desktop/`
2. 根据你的电脑芯片选择 "Download for Mac - Apple Chip"（M1/M2/M3芯片）或 "Intel Chip"
   - 不确定自己是哪种芯片：点屏幕左上角苹果图标 →"关于本机"，能看到芯片型号
3. 下载后双击安装包，把 Docker 图标拖进 Applications 文件夹
4. 打开 Applications 文件夹，双击 Docker 图标启动
5. 顶部菜单栏出现固定不动的小鲸鱼图标，说明启动成功

### 3.3 Linux (Ubuntu/Debian) 用户
打开终端，依次执行：
```bash
curl -fsSL https://get.docker.com | sh
sudo systemctl start docker
sudo systemctl enable docker
sudo usermod -aG docker $USER
```
执行完最后一行后，**注销并重新登录**（或重启一下），才能让当前用户免 `sudo` 使用 docker。

### 3.4 验证 Docker 装好了
打开终端（Windows 用户打开 "PowerShell" 或 "命令提示符"），输入：
```bash
docker --version
docker compose version
```
## 4. 第二步：获取项目代码

打开终端（Windows 用 PowerShell），依次执行：

```bash
# 1. 找一个你希望存放这个项目的位置，比如放在用户主目录下

# 2. 如果你有项目的压缩包(NovelCraft-部署包.tar.gz)，解压它

# 如果你是用 git 拉取代码，则用这个方式代替上面的 tar 解压：
# git clone <项目仓库地址>

```

解压/拉取完成后，用下面的命令确认目录结构对不对：
```bash
ls
```
你应该能看到 `backend`、`frontend`、`docker-compose.yml`、`.env.example` 这几个文件/文件夹。如果没看到，说明解压的位置不对，回到上一步检查。

## 5. 第三步：配置 `.env` 文件（最关键的一步）

`.env` 文件是整个系统的"总开关"，密码、密钥、AI 服务的 Key 都在这里配置。**这一步如果漏做或做错，系统会直接启动失败**，所以务必逐条核对。

### 5.1 复制模板文件

在项目根目录（`NovelCraft` 文件夹里）执行：

```bash
# macOS / Linux
cp .env.example .env

# Windows (PowerShell)
copy .env.example .env
```

### 5.2 生成需要的随机密钥

系统需要三个"强随机字符串"，用来保证密码和登录令牌的安全。**不要自己随便打几个字当密码**，用下面的命令生成：

**生成数据库密码：**
```bash
docker run --rm alpine sh -c "cat /dev/urandom | tr -dc 'A-Za-z0-9' | head -c 24"
```
执行后会输出一串随机字符，比如 `aB3xK9mQpL2vN8...`，复制它。

**生成 Redis 密码：** 再执行一次上面同样的命令（会得到不同的随机字符），复制它。

**生成 SECRET_KEY（登录令牌签名密钥）：**
```bash
docker run --rm python:3.11-slim python3 -c "import secrets; print(secrets.token_urlsafe(32))"
```
复制输出的字符串。

> 如果你的电脑上已经装了 Python，也可以直接用 `python3 -c "import secrets; print(secrets.token_urlsafe(32))"`，效果一样，不需要通过 docker 执行。

### 5.3 编辑 `.env` 文件

用记事本（Windows）、TextEdit（Mac，注意存成纯文本格式）、或者 VS Code 打开项目根目录下的 `.env` 文件，把下面每一项替换成你自己的值：

```ini
DB_USER=novelcraft
DB_PASSWORD=<粘贴你在5.2生成的第一个随机字符串>
DB_NAME=novelcraft

REDIS_PASSWORD=<粘贴你在5.2生成的第二个随机字符串>

SECRET_KEY=<粘贴你在5.2生成的第三个随机字符串>

ADMIN_USERNAME=admin
ADMIN_PASSWORD=<自己设置一个管理员密码，比如 MyStrongPass123>

DEEPSEEK_API_KEY=<粘贴你在"准备工作"里从DeepSeek官网复制的sk-开头的Key>
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat

ACCOUNT_ENCRYPTION_KEY=

CORS_ORIGINS=http://localhost:8080,http://localhost:3000,http://localhost:5173
```

**填写要点：**
- `<尖括号>` 里的内容（包括尖括号本身）要整个替换掉，不要把尖括号也留着
- `DB_PASSWORD`、`REDIS_PASSWORD`、`SECRET_KEY` 这三项**必须**是随机生成的字符串，不能留空，否则系统启动不了
- `ADMIN_PASSWORD` 建议至少 8 位，包含字母和数字
- `ACCOUNT_ENCRYPTION_KEY` 可以先留空，只有你要用"平台账号自动发布"功能时才需要填（见第 11 节之外的说明；用到的时候文档会告诉你怎么生成）
## 6. 第四步：启动系统

一切配置好之后，回到终端，确保当前目录是 `NovelCraft`（项目根目录，能看到 `docker-compose.yml` 的那个目录），执行：

```bash
docker compose up -d --build
```

- `-d` 表示在后台运行（不会占着这个终端窗口）
- `--build` 表示第一次启动时会构建镜像，**第一次执行会比较慢**（取决于网速，可能需要 5～15 分钟），因为要下载 Python、Node.js、PostgreSQL 等基础镜像并安装所有依赖包。请耐心等待，不要中途关闭终端。

看到类似下面的输出，没有报红色的错误，说明启动命令本身执行成功了（但还不代表系统真的能用，继续下一步验证）：

```
[+] Running 5/5
 ✔ Network novelcraft_default   Created
 ✔ Container novelcraft-db      Healthy
 ✔ Container novelcraft-redis   Started
 ✔ Container novelcraft-api     Started
 ✔ Container novelcraft-worker  Started
```

## 7. 第五步：验证部署是否成功

### 7.1 检查 4 个容器是否都在正常运行
```bash
docker compose ps
```
你应该看到 4 行，`api`、`worker`、`db`、`redis` 各一行，`STATUS` 列都是 `Up`（`db` 那一行可能显示 `Up (healthy)`）。

如果有某一行显示 `Exit` 或 `Restarting`，说明这个服务没启动起来，去看第 10 节的故障排查。

### 7.2 检查后端 API 是否响应正常
```bash
curl http://localhost:8100/health
```
应该返回：
```json
{"status":"ok","version":"8.0.0"}
```
如果这条命令报错"连接被拒绝"，说明 `api` 容器没起来，去看第 10 节。

### 7.3 检查任务队列 worker 是否真的在监听正确的队列
这一步很重要，之前的版本有过"worker 起来了但实际上没在干活"的问题，这里确认一下：
```bash
docker compose exec worker celery -A app.tasks.pipeline.celery_app inspect active_queues
```
输出里应该能看到 `idea`、`outline`、`chapter`、`review`、`publish`、`celery` 这几个队列名。如果这条命令报错或者队列列表是空的，说明 worker 配置有问题，去看第 10 节。

以上三步都通过，说明后端部分已经正确部署。

## 8. 第六步：打开系统，注册第一个账号

### 8.1 启动前端

如果你是在自己电脑上体验（不对外提供服务），前端也可以用 Docker 之外的方式简单跑起来。在项目根目录执行：

```bash
cd frontend
npm install
npm run dev
```

第一次执行 `npm install` 需要联网下载依赖包，完成后 `npm run dev` 会输出一个本地访问地址，通常是：

```
Local:   http://localhost:5173/
```

用浏览器打开这个地址。

> 如果你的电脑上没有装 Node.js：Windows/Mac 用户去 `https://nodejs.org` 下载安装 LTS（长期支持）版本，安装完重新打开终端再执行上面的命令。

### 8.2 注册账号

1. 打开的页面会自动跳转到登录页
2. 点击登录框上方的"**注册**"标签（页面有"登录/注册"两个标签可以切换）
3. 填写：用户名、邮箱、密码、确认密码
4. 密码要求：**至少8位，同时包含字母和数字**
5. 注册成功后会自动登录，进入"总控驾驶舱"首页

至此，部署完成，你可以开始使用了——具体每个功能怎么用，请看《使用手册》。

## 9. 日常操作：启动/停止/重启/看日志

以下命令都在项目根目录（能看到 `docker-compose.yml` 的目录）执行。

**停止系统（数据不会丢失）：**
```bash
docker compose stop
```

**重新启动（已经构建过，不需要重新加 `--build`）：**
```bash
docker compose start
```

**完全停止并删除容器（数据库数据仍然保留在 Docker 的存储卷里，不会丢）：**
```bash
docker compose down
```

**重新启动并应用代码/配置改动：**
```bash
docker compose up -d --build
```

**查看某个服务的实时日志（排查问题时最有用）：**
```bash
docker compose logs -f api        # 看后端API日志
docker compose logs -f worker     # 看后台任务日志
docker compose logs -f db         # 看数据库日志
```
按 `Ctrl+C` 退出日志查看（不会停止服务本身）。

**查看最近100行日志（不用一直盯着屏幕）：**
```bash
docker compose logs --tail=100 api
```

## 10. 常见问题排查

### 问题：`docker compose up` 报错，提示端口被占用
说明你电脑上 `8100` 端口已经被别的程序占用了。打开项目根目录下的 `docker-compose.yml`，找到 `api` 服务下的：
```yaml
ports:
  - "8100:8100"
```
把冒号前面的数字改成别的（比如 `"8200:8100"`），保存后重新执行 `docker compose up -d --build`，之后访问时把地址里的端口也改成新的。

### 问题：`docker compose ps` 里某个服务一直是 `Restarting`
执行 `docker compose logs <服务名>` 看具体报错信息，最常见的原因：
- `.env` 里的 `DB_PASSWORD`/`REDIS_PASSWORD`/`SECRET_KEY` 忘记填了，或者还留着 `.env.example` 里的占位符——回到第 5 节重新检查
- `DEEPSEEK_API_KEY` 格式不对（一定是 `sk-` 开头）

### 问题：AI 生成/审查/翻译功能提示"AI 服务暂时不可用"
- 检查 `.env` 里的 `DEEPSEEK_API_KEY` 是否正确、是否有多余的空格
- 登录 DeepSeek 官网确认账户余额是否充足
- 执行 `docker compose logs api` 看具体的错误信息

### 问题：批量生成/质量审查一直卡着不动
按第 7.3 节的方法确认 worker 是否真的在监听正确的队列。如果队列列表是空的：
```bash
docker compose restart worker
docker compose exec worker celery -A app.tasks.pipeline.celery_app inspect active_queues
```

### 问题：前端页面打开是空白的，或者一直转圈
1. 打开浏览器的开发者工具（按 F12），看 Console 标签里有没有报错
2. 常见原因是前端连不上后端 API，检查前端 `.env` 或 `vite.config.ts` 里配置的 API 地址是否指向了正确的后端端口

### 问题：忘记了管理员密码
管理员账号是首次启动时根据 `.env` 里的 `ADMIN_USERNAME`/`ADMIN_PASSWORD` 自动创建的。如果忘记了，直接修改 `.env` 里的 `ADMIN_PASSWORD` 不会更新已有账号的密码（只在首次创建时生效）。需要重新设置的话，最简单的方式是用第 8.2 节的注册流程再注册一个新账号使用。

## 11. 如果你要正式对外提供服务（不是自己一个人用）

如果你打算把这套系统部署到一台云服务器上、给多个人使用，除了前面的步骤，还必须额外做以下几件事，否则会有安全风险：

1. **`.env` 里的所有密码/密钥必须是真正随机生成的**（第 5.2 节的方法），绝对不能用 `.env.example` 里的占位符或者简单密码。
2. **给后端配一个真实域名 + HTTPS**，不要让用户直接用 IP 地址+端口访问。可以在前面加一层 Nginx 或者用 Caddy 自动申请 HTTPS 证书。
3. **修改 `.env` 里的 `CORS_ORIGINS`**，改成你真实的前端访问域名，例如：
   ```
   CORS_ORIGINS=https://your-domain.com
   ```
4. **数据库和 Redis 不要暴露公网端口**：本项目的 `docker-compose.yml` 默认已经不对外暴露这两个服务的端口，保持默认设置即可，不要为了图方便打开它们的端口映射。
5. **定期备份数据库**，见下一节。
6. **给云服务器配置防火墙**，只放行你需要对外提供的端口（通常是 80/443，如果直接用 8100 就放行 8100），其他端口一律关闭。

## 12. 升级到新版本 / 备份数据

### 12.1 备份数据库
```bash
docker compose exec db pg_dump -U novelcraft novelcraft > 备份_$(date +%Y%m%d).sql
```
这会在当前目录生成一个 `.sql` 备份文件，建议定期执行并把备份文件保存到别的地方（比如云盘），不要只放在同一台服务器上。

### 12.2 从备份恢复
```bash
cat 备份_20260101.sql | docker compose exec -T db psql -U novelcraft novelcraft
```

### 12.3 升级到新版本代码
```bash
# 1. 先备份数据库(见12.1)
# 2. 获取新版本代码，覆盖当前目录(注意不要覆盖你的 .env 文件)
# 3. 重新构建并启动
docker compose down
docker compose up -d --build
```
