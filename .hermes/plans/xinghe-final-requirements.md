# 星禾写作助手 — 终极需求文档

> 对标：番茄作家助手 / 阅文作家助手 / Sudowrite
> 架构：自研，零框架依赖。Web（移动+桌面）+ 移动端 PWA

---

## 一、产品定位

AI 驱动的全流程小说写作平台。覆盖创作、管理、分析、发布全链路。

---

## 二、功能需求（共 40 项）

### 2.1 书库管理（6 项）
| # | 需求 | 说明 |
|---|------|------|
| 1 | 书库列表 | 卡片网格展示所有作品，显示书名/题材/章节数/字数/封面色 |
| 2 | 新建书籍 | 底部弹窗输入书名+题材，即时创建 |
| 3 | 删除书籍 | 长按或按钮删除，带确认弹窗 |
| 4 | 搜索/筛选 | 按书名或题材筛选 |
| 5 | 写作进度 | 每本书显示完成百分比进度条 |
| 6 | 批量操作 | 批量删除/导出 |

### 2.2 书籍详情（8 项）
| # | 需求 | 说明 |
|---|------|------|
| 7 | 书籍头图 | 渐变色封面 + 书名 + 统计标签 |
| 8 | 章节列表 | 按序排列，显示标题/字数/评分/状态 |
| 9 | 全书总纲展示 | Markdown 格式，可复制 |
| 10 | 小说简介 | 自动拼接书名+题材+总纲 |
| 11 | 世界观设定 | 独立卡片展示 |
| 12 | 人物系统展示 | 每个角色的姓名/身份/特质/状态/位置/目标/关系 |
| 13 | 伏笔追踪面板 | 待回收/已回收伏笔，埋设章节/回收章节 |
| 14 | 防崩检测面板 | 一致性问题章节 + 问题标签 |

### 2.3 知识库（3 项）
| # | 需求 | 说明 |
|---|------|------|
| 15 | 术语表管理 | 术语+定义+分类，表格形式，可增删改 |
| 16 | 修炼/力量体系 | 自由文本编辑 |
| 17 | 世界规则 | 自由文本编辑 |

### 2.4 章节生成（5 项）
| # | 需求 | 说明 |
|---|------|------|
| 18 | 续写下一章 | 自动组装上下文（总纲+前5章摘要+人物+伏笔+防崩提醒）→ 大纲 → 正文 → 摘要 → 审查 |
| 19 | 全自动扫榜→成书 | 选平台/题材 → 扫榜分析 → 全书规划 → 黄金三章一次性生成 |
| 20 | 拆文仿写 | URL或粘贴原文 → 自动拆解文风/节奏/爽点 → 仿写新章 |
| 21 | 创意生成大纲 | 输入一个 idea → AI 生成全书总纲+章节树 |
| 22 | 批量生成 | 设置总章数，自动逐章生成直到完成，支持断点续传 |

### 2.5 记忆系统（3 项）
| # | 需求 | 说明 |
|---|------|------|
| 23 | 章节记忆 | 每章存150字摘要，续写时注入最近5章 |
| 24 | 人物记忆 | JSON 追踪人物状态，每章自动更新 |
| 25 | 伏笔追踪 | 埋设/回收伏笔列表，续写时注入待回收项 |

### 2.6 防崩系统（2 项）
| # | 需求 | 说明 |
|---|------|------|
| 26 | 一致性校验 | 5维检测（人物/时间线/地点/风格/重复信息） |
| 27 | 防崩提醒 | 近3章问题作为警告注入续写上下文 |

### 2.7 审查系统（3 项）
| # | 需求 | 说明 |
|---|------|------|
| 28 | 7维度评分 | consistency/ai_trace/pacing/character/hook/dialogue/ending |
| 29 | 评审意见 | 具体问题 Tag + AI痕迹 Tag |
| 30 | 审查建议与改进方向 | 章节页合并展示评分+问题+改进 |

### 2.8 章节阅读（4 项）
| # | 需求 | 说明 |
|---|------|------|
| 31 | 正文阅读 | 衬线字体，舒适行距，字号可调 |
| 32 | 优化改写 | 保留核心事件，增强文笔 |
| 33 | 重新审查 | 重新跑7维度审查 |
| 34 | 一键复制 | 复制全文 |

### 2.9 分析看板（3 项）
| # | 需求 | 说明 |
|---|------|------|
| 35 | 日更统计 | 字数柱状图 + 趋势折线图 |
| 36 | 评分分布 | 饼图/柱状图 |
| 37 | 写作日历 | 热力图 |

### 2.10 标准化（3 项）
| # | 需求 | 说明 |
|---|------|------|
| 38 | 书名自动提取 | 从 LLM 输出中智能提取，不出现"AI生成" |
| 39 | 章名自动提取 | 从正文 `# 第X章` 提取 |
| 40 | 禁止元文本 | 正文不出现"上一章""本章"等级述词 |

## 三、技术架构

### 3.1 整体架构
```
┌────────────────────────────────────┐
│         星禾写作助手                │
├──────────┬──────────┬──────────────┤
│ Web (移动)│ Web (桌面)│ PWA (手机安装)│
│ <768px   │ ≥768px  │ standalone   │
├──────────┴──────────┴──────────────┤
│         单页 HTML/CSS/JS           │
│         零框架，零构建              │
├────────────────────────────────────┤
│     REST API → VPS :8100          │
├────────────────────────────────────┤
│   FastAPI + PostgreSQL + DeepSeek  │
└────────────────────────────────────┘
```

### 3.2 前端技术栈
```
HTML5 + CSS3 (Grid/Flexbox/Animations)
Vanilla JavaScript (ES2020+)
零依赖：无 React/Vue/Flutter/任何框架
Service Worker (PWA 离线缓存)
Web App Manifest (添加到主屏幕)
CSS Custom Properties (主题切换)
```

### 3.3 响应式策略
```
移动端 (<768px)：
  - 单列布局
  - 底部 Tab 导航
  - 卡片竖向排列
  - FAB 浮动按钮
  - 底部弹出 Modal
  - 触摸友好的最小 44px 点击区域

桌面端 (≥768px)：
  - 侧边栏导航
  - 两列/三列布局
  - 悬停效果
  - 对话框式 Modal
  - 键盘快捷键
```

### 3.4 后端（已完成）
```
FastAPI :8100
PostgreSQL (复用 AI Workbench DB)
DeepSeek API (SSH 中继)
17 个 REST 端点
22+22 字段 DB Schema
```

## 四、UI 设计规范

### 4.1 设计语言
```
对标：番茄作家助手 / Apple Design / Material You
风格：温暖、专业、中文优先
色彩：主色 #FF6B35（橙红）渐变 #FF2442
背景：#F8F9FA（浅灰）
卡片：纯白 + 12px圆角 + 轻微阴影
字体：系统默认中文（PingFang SC / Microsoft YaHei）
```

### 4.2 组件规范
```
卡片：白色背景，16px圆角，2px阴影
按钮：12px圆角，主按钮填充色，次按钮描边
输入框：12px圆角，聚焦时橙色描边
标签：20px圆角，半透明背景
模态框：底部弹出(移动)，居中弹出(桌面)
导航栏：底部4Tab(移动)，左侧侧边栏(桌面)
```

### 4.3 动画
```
页面切换：200ms fade
卡片点击：轻微缩放
模态框弹出：slide-up 300ms
列表加载：staggered fade-in
```
